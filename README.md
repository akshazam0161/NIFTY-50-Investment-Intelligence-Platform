# NIFTY-50 Investment Intelligence Platform

A Streamlit dashboard that turns raw NIFTY-50 historical market data into something actually useful — risk metrics, directional predictions, portfolio recommendations, and anomaly detection, all in one place.

Built for the Data-Driven Investment Intelligence challenge using the [Kaggle NIFTY-50 dataset](https://www.kaggle.com/datasets/rohanrao/nifty50-stock-market-data/data).

---

## What it does

The app has four main sections:

**Performance Visualizer** — plots closing price with Bollinger Bands for any selected ticker, alongside five risk metrics: cumulative return, annualized volatility, Sharpe ratio, Sortino ratio, and maximum drawdown.

**ML Predictive Engine** — trains a Random Forest classifier to forecast whether a stock closes up or down the next day. Uses 5-fold `TimeSeriesSplit` cross-validation so no future data leaks into training. Feature importances are shown as a bar chart so you can see what the model is actually using.

**Portfolio Construction** — builds a 3-stock portfolio for three investor profiles (Conservative, Balanced, Aggressive) using inverse-volatility weighting. Conservative = lowest volatility stocks, Balanced = highest Sharpe ratio, Aggressive = highest cumulative return. Also shows a pairwise return correlation heatmap across the selected portfolio.

**Anomaly Detection** — flags trading days where the daily return exceeded ±3 standard deviations of the stock's own historical mean. These show up as red dots on the price chart with a full table of dates and return values.

## Setup

### 1. Clone the repo

```bash
git clone https://github.com/<your-username>/<your-repo>.git
cd <your-repo>
```

### 2. Create a virtual environment

```bash
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Get the data

Download the NIFTY-50 dataset from Kaggle:  
https://www.kaggle.com/datasets/rohanrao/nifty50-stock-market-data/data

Extract it so the individual stock CSVs (one per ticker, e.g. `RELIANCE.csv`, `TCS.csv`) are inside a `data/` folder at the project root. The dataset also contains a `stock_metadata.csv` — that file is fine to leave in the folder, `combine_data.py` skips it automatically.

### 5. Combine the per-stock files

The app expects a single merged file called `NIFTY50_all.csv`. Run the combine script once:

```bash
python combine_data.py data/
```

This reads every stock CSV in `data/`, adds a `Symbol` column inferred from the filename, concatenates everything, and writes `data/NIFTY50_all.csv`. You should see output like:

```
[combine_data.py] Scanning: /path/to/data/

  [+] ADANIPORTS.csv   → Symbol: ADANIPORTS    (4,142 rows)
  [+] ASIANPAINT.csv   → Symbol: ASIANPAINT    (4,142 rows)
  ...
  [✓] Merged 50 file(s) → 207,100 total rows, 50 unique symbols
```

### 6. Run the app

```bash
streamlit run app.py
```

The app opens at `http://localhost:8501`.

## How the pipeline works

`data_pipeline.py` runs in a fixed sequence every time the app loads (result is cached via `@st.cache_data`):

1. Drops rows with missing dates, symbols, or close prices
2. Removes days where the single-day return exceeded +200% or fell below -70% — these are almost always unadjusted stock splits or data errors
3. Filters out rows with volume under 100 to remove non-trading days
4. Computes SMA (5, 20, 50), EMA (5, 20), Bollinger Bands with bandwidth, RSI (14-day Wilder smoothing), MACD with signal and histogram, 20-day rolling volatility, ROC-10, and volume ratio
5. Generates the `Target_Dir` label by shifting daily returns one day forward per symbol using `np.where` (not `.apply()`) to prevent NaN rows from being silently converted to 0
6. Drops the lookback margin rows created by `SMA_50` and `Volatility_20`, and the last row per symbol that has no valid target

## Notes on model performance

The Random Forest classifier typically achieves around 50–52% cross-validation accuracy on most NIFTY-50 stocks. That's close to random, which is expected — if a simple technical-indicator model could reliably call direction on large-cap NSE stocks, the edge would get arbitraged away quickly. The point of the ML section is the framework: no lookahead bias, time-ordered validation, and transparent feature attribution via Gini importance. Whether the model has real predictive power on out-of-sample future data is a separate question.

Precision on "UP" calls varies by stock but can dip below 50% on individual tickers. The technical report discusses this in more detail.

## Reproducing results

Training is non-deterministic across machines even with `random_state=42` set, because scikit-learn's tree building can vary slightly depending on the NumPy/threading environment. CV accuracy will typically land within ±1–2% of what's shown in the screenshots. Portfolio allocations are fully deterministic given the same input data.
